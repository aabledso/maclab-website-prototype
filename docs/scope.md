# Project Scope Statement
## Project Name: MacLab Website Prototype
## Project Purpose
The purpose of this project is to design and develop a professional, user-friendly
website for MacLab, a fictional fast food restaurant focused on macro-friendly and
healthy meals. The website will allow users to learn why MacLab was created and view menu items.
## Description
This project revolves around the Final Project requirements of creating a two page
prototype for a website that includes a stylesheet, project scope statement,
scope.md, plan.md, and retrospective.md files. The website will have a homepage
introducing MacLab and a menu page.
## Desired Results
The project will be deemed successful when the following outcomes are achieved:
* A functional two page website prototype is completed
* Website has a home page and menu page
* Website is visually appealing and well organized
* Code is well organized and functions correctly
* All files are stored in a public GitHub repository that is professional and
includes Read Me file
* Repository includes Scope, Plan, and Retrospective Markdown files
* Repository's root directory includes index.html, about.html, and style.css
## Exclusions
* No customer logins, passwords, or authentication methods
* No online ordering, payments, or shopping carts
* No databases or advanced functionality
* No JavaScript
* No custom domain
* No advanced graphics
* No pages other than Home and Menu
## Feedback Loops
Feedback will be provided by the course instructor throughout the development of
the website. Feedback will be used to improve different aspects of the website and
to resolve any issues as they arise. Informal self-review and testing will also be
conducted throughout development to ensure the website functions as expected. I
will periodically reflect on what I did well, what I struggled with, and develop
plans for improvement. This will create an internal feedback loop that will allow
me to continuously improve.
## People Resources
The primary team member is myself. I will act as the project manager, designer, and
developer. The course instructor will serve as the key stakeholder and reviewer.
They will provide feedback and approve the final project.
